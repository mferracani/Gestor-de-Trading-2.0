# Front-End Design — Production-Grade UI Skill

## Purpose

Generate frontend code that looks designed by a human, not generated by AI. Avoid the telltale signs of AI-generated UIs: generic blue palettes, centered everything, default shadows, predictable layouts.

## Design Principles

### 1. Typography First
- Choose distinctive font pairings (not just system fonts)
- Use scale hierarchy: display > heading > subheading > body > caption
- Apply proper line height (1.4-1.6 for body, 1.1-1.2 for headings)
- Use font weight to create hierarchy, not just size

### 2. Color with Character
- Avoid pure black (#000) — use near-blacks (#1a1a2e, #0f0f0f)
- Avoid default blue — choose a palette that fits the content
- Use color sparingly for emphasis, not decoration
- Ensure sufficient contrast (WCAG AA minimum)
- Consider dark/light mode from the start

### 3. Layout That Breathes
- Use generous whitespace — when in doubt, add more
- Break the grid occasionally for visual interest
- Use asymmetric layouts where appropriate
- Align elements intentionally (not everything centered)
- Use max-width for content readability (65-75 characters per line)

### 4. Motion and Interaction
- Add subtle transitions (200-300ms) for state changes
- Use hover states that feel responsive
- Consider scroll-based animations sparingly
- Loading states should feel crafted, not default

### 5. Details That Matter
- Custom focus styles (not browser defaults)
- Proper spacing between sections (use a spacing scale)
- Icons that match the visual style
- Consistent border radius (pick one and stick to it)
- Subtle gradients or textures instead of flat colors

## Anti-Patterns to Avoid
- ❌ Blue + white + shadows = "AI starter kit"
- ❌ Centered everything with equal padding
- ❌ Generic hero sections with "Welcome to [Product]"
- ❌ Cards with identical spacing and shadows everywhere
- ❌ Default Tailwind utilities without customization
- ❌ Rainbow gradients on buttons
- ❌ Stock photo heroes

## Tech Stack Preferences
- **React/Next.js** with Tailwind CSS as default
- **Shadcn/UI** for component base when appropriate
- **Framer Motion** for animations
- **Lucide** or **Phosphor** for icons
- Responsive by default (mobile-first)

## Workflow
1. Understand the purpose and audience of the interface
2. Choose a visual direction (modern, minimal, bold, editorial, etc.)
3. Define the color palette and typography before coding
4. Build the layout structure first, then add details
5. Test at multiple breakpoints
6. Review for accessibility

## Rules
- Every component must be responsive
- Include hover, focus, and active states
- Use semantic HTML
- Export clean, maintainable code
- Include comments explaining non-obvious design choices
